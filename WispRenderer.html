Wisp - Realtime Raytracing

A real time hybrid raytraced rendering plugin replacing the Autodesk Maya viewport leveraging the power of DirectX Raytracing

  • Role: Graphics Programmer
  • Team members: 9
  • Project Length: 32 Weeks
  • Period: September 2018 - July 2019
  • Responsibilities
    • Raytraced Ambient Occlusion
    • Graphics Settings

Wisp is a general purpose high level rendering library. Specially made for real-time raytracing with NVIDIA RTX graphics cards. Made by a group of students from Breda University of Applied Sciences specializing in graphics programming. This rendering library is used to power a Maya plugin which bring real-time hybrid raytracing to the modeling viewport to bridge the gap between the preview and the final rendering. Our hybrid renderer uses raytracing for Ambient Occlusion, Shadows and Reflections. On top of this, Wisp also features a PBR material preview renderer based on Unreal Engine 4's implementation of physically based rendering.

My Contribution

I started working on this project since I wanted to improve my skill as a graphics programmer and was looking forward to a tech focused project. In the initial stages of the project I've been working on implementing the basics of a PBR pipeline into our deferred rasterized renderer. During this time I've also worked on some general engine maintenance to improve the quality of life during future development. Later on I started working on implementing Raytraced Ambient Occlusion during which my understanding of both DirectX 12 increased significantly. In the final stages of the project I've implemented crash logging and switched back and forth between iterating on the Ambient Occlusion render task and creating an editor for the graphics settings to tweak the render tasks during run time.
